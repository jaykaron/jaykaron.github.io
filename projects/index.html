AAW

Built with Nadav Hames for a 4th year CS course, the Algorithmics Animation Workshop (AAW) is a website that lets users visualize and interact with many famous algorithms.

Hosted by York University's CS department, it features an easy-to-use JS library built with D3 to allow other students to continue to develop and add algorithm visualizations.

This Website

After learning a little bit of HTML, CSS and JavaScript in high-school and university, I wanted to hone my skills and get some experience using some newer web technologies.

I started out using only vanilla jQuery and SASS, but after spending a considerable amount of time working on this cool nav-bar (please excuse the color scheme), I asked myself "Was that really worth it?" and switched to Bootstrap.

With Bootstrap it takes about 2 minutes to make the nav-bar at the top of this page which works across browsers and has mobile support. So that definitely sped things up.

After that it was pretty smooth sailing, all that was required was time. Time for going through Bootstrap's documentation, time for fiddling with the CSS, etc.

2019 Update

I was tired of all the copy and pasted code in HTML so I started using Nunjucks, a JS templating engine and Gulp to spit out the static site. This has reduced the LOC signifcantly and has made it fun to update the website again.

Wikipedia Graph

A joint attempt with my brother to graph the links of Wikipedia and find out something usesful.

So far we have found out that their are lots of links. We are working on sorting them out.

We are using Python to scrape Wikipedia and putting the results into a Neo4J database, a system for graph databases that has a special query language called Cypher.

You can see the existing code in the Github repository. Here's a picture of a lot of the articles connected to Science and the links between them to keep you busy.

Update

As we were working on this project, Jacob Wenger made an beautiful site, Six Degrees of Wikipedia, that does something similar to what we were envisioning. In addition, processing was taking us forever because we were working on consumer laptops. Due to those two factors we decided to cancel the project.
